可以看出，高级语言是最适合人写的，汇编语言和机器语言十分反人类，所以计算机飞速发展的今天，几乎所有的还在使用的语言都是高级语言

高级语言的运行机制

因为高级语言是用底层语言写的，所以高级语言不能直接被计算机读取。
目前有两种办法让计算机读取

编译型

有很多语言是将高级语言进行转换，转换为机器语言。这个过程为编译

  • 优点
    1. 不需要高级语言的运行环境，用户可以直接运行
    2. 运行效率高
  • 缺点
    1. 无法修改已经编译好的程序
    2. 无法直接移植平台
  • 例子
    • C/C++

解释性

有的语言则是不做任何事情，在需要运行的时候直接把源代码一条一条地解释运行。

  • 优点
    1. 不需要编译，省掉了许多麻烦
    2. 基本上可以直接跨平台使用
    3. 修改程序十分简单而轻松
  • 缺点
    1. 运行时需要运行环境
    2. 运行效率低下
  • 例子
    • PHP
    • Python
    • JavaScript

Java的运行原理

Java想像解释性语言可以跨平台又想像编译型语言效率高，便采用了一种折中的办法。

  1. 将源代码编译生成为.class文件，这个文件是与平台无关的字节码文件。
  2. .class文件用JVM运行，不同的平台只需要用不同平台的JVM就可以直接运行.class文件。

Java的JVM

  1. Q：jvm是什么

    A：JVM为Java的虚拟机，不同的平台用不同的虚拟机，但他对于的.class文件的运行方式是相同的，从而达到跨平台功能

    这里有个形象的比喻：

    电脑（.class程序）如果要连接手机（macOS）、相机（Windows）等，必须要用转接连接线（JVM）才能连接到手机、相机等（系统）。而电脑(.class程序)不知道手机、相机等的连接方式
  2. Q：jvm为什么下载不到

    A:JRE已内置JVM，请到Java官方下载

    jvm仅仅只是虚拟机，在平台上不能单独运行，还要有JRE的其他支持诸如文件效验之类的。

字节码文件

计算机是由无数个0和1组成的，通过有规定的编码进行排列的

举个例子：我们如下规定：

编码 含义 编码 含义
00000001 的 00000010 逼
00000011 世 00000100 真
00000101 界 00000110 牛
00000111 我

现在我们的文件保存的数据是
00000111 00000001 00000011 00000101 00000100 00000110 00000010
那么逐个转换之后就是我的世界真牛逼（打广告）

OK，看了这个就懂了计算机为什么只需要0和1了吧。
事实上，计算机以8位0和1的组合作为一个最小的单位，为「字节」，
我们将每一个0或者1称之为「位」。所以字节码文件就是以字节为单位的文件。
